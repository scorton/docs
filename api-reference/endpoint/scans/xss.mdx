## XSS Scan

The `xss` endpoint probes web inputs and response reflections to flag potential cross-site scripting risks.

---

## Endpoint

`POST https://api.scorton.tech/scan/xss`

### Authentication

Authorization: `Bearer <YOUR_API_KEY>`.

### Request Body

```json
{
  "url": "https://example.com/search?q=foo",
  "payloads": ["<script>alert(1)</script>", "\" onmouseover=alert(1) \""],
  "followRedirects": true,
  "timeoutMs": 15000
}
```

### Response (200)

```json
{
  "url": "https://example.com/search?q=foo",
  "issues": [
    {
      "type": "reflected",
      "parameter": "q",
      "evidence": "<script>alert(1)</script>",
      "severity": "high"
    }
  ],
  "testedPayloads": 2,
  "scanAt": "2025-10-13T14:48:40Z"
}
```

### Error (429)

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Too many scan requests; retry later"
  }
}
```

### cURL Example

```bash
curl -X POST 'https://api.scorton.tech/scan/xss' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://example.com/search?q=foo",
    "payloads": ["<script>alert(1)</script>", "\\" onmouseover=alert(1) \\\""],
    "followRedirects": true,
    "timeoutMs": 15000
  }'
```

### Verification

- Review `issues` for parameters and severity.
- Confirm `testedPayloads` matches the number of payloads sent.
